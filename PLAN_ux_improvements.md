## 1. 온보딩 개선

### 1-1. 테마 선택 화면

**변경 사항**

- 버건디/세이지의 서브텍스트(`고급스러운`, `자연스러운`) **완전 제거** → 컬러 카드 자체로 감성 전달
- 버건디/세이지 아이콘을 라벤더/테라코타와 동일하게 **하트 이모지(♥)** 형태로 통일
- 프리미엄 테마 배지: 주황 배경 패딩 제거 → 아이콘만 간결하게 표시

**프리미엄 테마 프리뷰 + 결제 유도 플로우**

1. 무료 유저도 라벤더/테라코타 선택 시 → 테마 프리뷰 즉시 적용 (홈화면이 해당 컬러로 변함)
2. 해당 상태에서 [다음] 버튼 탭 → Bottom Sheet 팝업 노출
   - 내용: "이 테마는 프리미엄 전용이에요 💎 지금 업그레이드하면 모든 테마를 자유롭게 사용할 수 있어요"
   - CTA: `프리미엄 시작하기` / `기본 테마로 계속`
3. `기본 테마로 계속` 선택 시 → 버건디로 복귀 후 다음 단계 진행

### 1-2. 셋로그 기능 소개 온보딩 (최앞단 추가)

기존 온보딩 앞에 **Spenlog 핵심 기능 소개 슬라이드** 추가. 필라이즈의 기능 설명 프로세스 레퍼런스.

**슬라이드 구성 (3장)**

| 슬라이드 | 헤드카피 | 서브카피 |
|---------|---------|---------|
| 1 | '스타벅스 육천원 카드' 한 줄이면 | AI가 자동으로 분류해줘요 |
| 2 | 이번 달 지출 현황을 한눈에 | 카테고리별 예산 달성률 확인 |
| 3 | 계좌·카드·고정비 연결하면 | 잔액이 자동으로 반영돼요 |

- 각 슬라이드: 실제 앱 화면 스크린샷 배경 + 하단 텍스트 카드
- 우측 상단 `건너뛰기` 버튼 항상 노출

### 1-3. 홈화면 온보딩 하이라이트 UI 개선

**현재 문제**: 안내 텍스트가 너무 작아 인지 어려움

**개선 방향**
- 하이라이트 영역 외 배경: 반투명 다크 오버레이 (`rgba(0,0,0,0.55)`)
- 화면 중앙에 반투명 흰색 텍스트 래퍼 (배경 `rgba(255,255,255,0.92)`, 보더 라디우스 16px)
- 손글씨 스타일 화살표로 하이라이트 영역 직접 지칭
- 텍스트 크기: 기존보다 2단계 상향

**하이라이트 순서 수정**

| 단계 | 포커스 영역 | 안내 문구 |
|-----|-----------|---------|
| 1/4 | AI 입력 박스 (상단 두 번째) | "여기서 지출을 기록해요. '스타벅스 육천원 카드' 한 줄이면 AI가 자동 분류!" |
| 2/4 | 홈 커버 배너 | "홈 화면을 내 취향으로 꾸밀 수 있어요" |
| 3/4 | 내역 탭 | "모든 지출·수입 내역을 여기서 확인해요" |
| 4/4 | 리포트 탭 | "월간 소비 패턴을 보고 AI와 함께 다음 달을 계획해요" |

### 1-4. 자산 관리 탭 최초 진입 가이드

**트리거**: 최초 진입 시 무조건 노출 (localStorage `spenlog_asset_guide_shown` 플래그)

**가이드 슬라이드 (4단계)**

| 단계 | 내용 |
|-----|-----|
| 1 | 계좌 등록 → 잔액 자동 추적 |
| 2 | 카드에 계좌 연결 → 결제 시 잔액 자동 차감 |
| 3 | 고정비·적금에 계좌 연결 → 루틴 기록 시 잔액 자동 반영 |
| 4 | CTA: "지금 계좌 먼저 입력해볼까요?" → [계좌 추가] 버튼으로 직접 연결 |

## 2. 홈화면 개선

### 2-1. 카테고리 섹션 리네임 및 금액 정책

- **섹션명**: `카테고리별 지출 현황` → `카테고리 현황`
- **금액 표시**: 해당 월 해당 카테고리 **총 합계** (수입·지출 구분 없이 net)
  - 예) 생활비에서 지출 10,000원 + 생활비 수입 300,000원 = **+290,000원** (+ 표시)
  - 지출만 있으면 그대로 금액 표시, 수입이 섞이면 `+` 프리픽스로 시각 구분
- **표시 카테고리**: 유저가 활성화한 카테고리 전체 (수입 포함)

### 2-2. 홈화면 인플레이스 편집 (네이버 블로그 스타일)

**현재**: 편집 아이콘 탭 → 모달 팝업 오버레이

**변경**: 홈화면 자체에 편집 오버레이 적용 — 실제 홈 콘텐츠를 보면서 편집

**UX 플로우**
1. 홈 우상단 팔레트 아이콘(🎨) 탭 → 편집 모드 진입
2. 홈화면 위에 편집 오버레이 렌더링:
   - 커버 이미지 영역: `📷 이미지 변경` 탭 버튼 오버레이
   - 카테고리 그리드: 각 카드 우상단에 `✏️` 핸들 노출, 드래그로 순서 변경
   - 각 섹션 좌측에 `⠿` 드래그 핸들 표시
3. 상단 고정 바: `취소` (좌) · `홈편집` (중앙, 비활성) · `적용` (우)
4. 변경 없이 취소 → 원상복귀 / 적용 → DB 저장 + 편집 모드 해제

**편집 가능 항목**
- 커버 이미지 (갤러리/URL)
- 커버 스타일 (그라디언트 컬러 선택)
- 카테고리 그리드 4칸 이미지/아이콘 교체
- 카테고리 순서 드래그 재배치

**아이콘 변경**: 연필 → 🎨 팔레트 (또는 슬라이더 아이콘)

### 2-3. 홈화면 편집 즉각 동기화

편집 후 `적용` 시 홈화면 즉각 반영 (현재 새로고침 필요한 경우 제거). `router.refresh()` 또는 로컬 state 직접 업데이트.

## 3. 카테고리 관리 분리

### 3-1. 독립 페이지 `/category` 신설

**접근 경로**
- 홈화면 `카테고리 현황` → `전체 보기` 탭
- 자산 탭 → 예산 섹션 → 카테고리 관리
- 설정 탭 → 카테고리 관리 (추가)

**기능**

| 기능 | 내용 |
|-----|-----|
| 카테고리 목록 | 현재 유저 카테고리 전체 표시 (순서 포함) |
| 추가 | `+ 카테고리 추가` → 이름 입력 |
| 편집 | 이름 인라인 수정 |
| 삭제 | 즉시 완전 삭제 (복원 토스트 제거) |
| 드래그 재정렬 | 터치 드래그로 순서 변경 |
| 기본 카테고리 | 생활비/활동비/고정비/친목비/예비비/수입 — 편집·삭제 모두 허용 |

**전역 동기화 포인트**
- 카테고리 변경 → 홈 그리드, 예산 탭, 지출 입력 폼, 내역 필터 전부 즉각 반영
- 구현 방식: Supabase Realtime 또는 `router.refresh()` + 로컬 state 업데이트

## 4. 자산 탭 개선

### 4-1. 예산 설정 구조 개선 (ON/OFF 토글)

**변경 사항**

- 현재: 모든 카테고리 항상 표시 → 설정 압박
- 변경: 카테고리별 **ON/OFF 토글** 추가
  - OFF 카테고리: 회색 처리, 예산 달성률 계산 제외
  - ON 카테고리만 입력 필드 활성화
- 카테고리 목록: 유저가 `/category`에서 설정한 카테고리와 자동 동기화
  - 카테고리 추가/삭제 시 예산 탭 즉각 반영
- AI 스마트 추천도 ON 카테고리에만 금액 배분

### 4-2. 루틴 기록 즉각 동기화 ✅ (구현 완료)

루틴 완료 시 `RoutineBanner → AssetsClient` 콜백(`onAccountsChange`)으로 계좌 잔액 즉각 반영. 새로고침·탭 이동 불필요.

### 4-3. 계좌/현금 현금 추가 동선

**현재**: 현금 추가 동선 불명확

**변경**: 계좌 섹션 하단에 `💵 현금 추가` 숏컷 버튼 추가
- 탭 시 유형='현금', 이름='현금'이 프리셋된 미니 폼 노출
- 잔액만 입력하면 바로 등록

### 4-4. 루틴 탭 기본 상태 변경

- 루틴 탭은 항상 **기본 ON(활성화)** 상태
- 단, 등록된 루틴이 없으면 **접힌 상태(Collapse)** 로 노출
- 루틴이 생성되면 **펼쳐진 상태(Expand)** 로 자동 전환

### 4-5. 카테고리 [전체보기] 동선 변경

- 기존: 내역 탭으로 이동 (최근 지출 전체보기와 중복)
- 변경: `/category` 카테고리 관리 화면으로 바로 진입

## 5. AI 입력 실패 플로우 개선

**현재 문제**: 실패 시 에러 메시지 1초 노출 → 바로 직접 입력 화면 전환 (인지 어려움)

**변경 플로우**

1. AI 분류 실패 → **Modal 팝업** 노출
   - 제목: "인식에 실패했어요"
   - 내용: "현재 AI 인식이 불안정해요. 직접 입력해 주세요."
   - 버튼: `직접 입력하기`
2. 버튼 탭 또는 2초 후 자동 → 직접 입력 화면으로 자연스럽게 전환
3. AI 지속 실패 시 원인 분석 필요 (Gemini API 키 만료 / rate limit 확인)

## 6. 즉각 동기화 요구사항 (개발 전반)

아래 액션 발생 시 관련 UI 전체 즉각 반영 (새로고침 불필요):

| 액션 | 즉각 반영 대상 |
|-----|-------------|
| 루틴 완료 | 계좌 잔액, 이번달 지출 합계 |
| 카테고리 추가/수정/삭제 | 홈 그리드, 예산 탭, 지출 입력 폼 |

| 계좌 잔액 변동 | 자산 탭 총 잔액, 홈 배너 |
