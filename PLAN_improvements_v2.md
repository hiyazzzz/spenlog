## 즉시 구현 가능 항목

### 1. 금액 표기 전역 변경
- `₩3,000,000` → `3,000,000원`
- `formatCurrency()` 함수 수정 → 전역 반영
- 영향 범위: 홈, 자산, 예산, 내역, 리포트 전체

### 2. 탭 이모지 제거
- BottomNav 아이콘 이모지 제거 (홈/내역/자산/리포트/설정)
- 자산 탭 섹션 이모지 제거 (💰, 🎯, 🏦, 💳, 📌)
- SVG 아이콘 또는 텍스트만 사용

### 3. 홈화면 편집 — 오버레이 방식으로 전환
- 현재: 편집 버튼 → 전체화면 이동
- 변경: 실제 홈화면 위에 편집 오버레이 (이미지 변경 자유 → 적용 시 프리미엄 게이트)
- 편집 아이콘/이미지변경 아이콘: 심플 연필 벡터로

### 4. 카테고리 관리 페이지 (/category) 수정
- **버그**: 기본 카테고리(생활비, 고정비 등)가 표시 안 됨 → 수정 필요
- DB `categories` 테이블에 기본 카테고리가 없으면 seed 또는 런타임에 보완
- 카테고리 순서 1~4위 → 홈 그리드 카테고리 현황에 반영

### 5. 홈 카테고리 현황 카드 디자인 개선 (A안)
- 이모지 제거
- 카테고리별 테마 컬러 기반 그라디언트 배경 + 카테고리명 + 금액 표시
- DB `categories.color` 컬럼 추가 (사용자가 색상 지정)
- "전체 보기 →" → "카테고리 관리 →" 워딩 변경

### 6. 예산 탭 수정
- AssetsClient 예산 섹션에서 `CategoryManager` 컴포넌트 제거
- 예산 항목 = 카테고리 관리 동기화 (이미 customCategories prop으로 연결됨)
- AI 예산 추천 무료화 (premium 체크 제거)

### 7. AI 예산 추천 개선
**단일 "AI 추천" 버튼으로 통합:**
- 프론트: 항상 "✨ AI 추천" 버튼 하나 표시
- 백단 로직:
  - 3개월 데이터 없음 → "보통" 퍼센테이지(수입의 25% 저축) 기반 추천
  - 3개월 데이터 있음 → Gemini 패턴 분석 추천
- 오차 안내: 데이터 3개월 미만 시 "데이터가 부족해 오차가 있을 수 있어요" 표시
- Gemini 프롬프트에 유저 커스텀 카테고리 이름 동적으로 주입

### 8. 저축 목표 달성 기준 (C안)
**달성 = 고정저축 + 추가 절약분 합산**

계산식:
- 고정저축 달성액 = `savings_payments.is_paid = true` 합계 (이번달)
- 추가 절약분 = `이번달 수입 - 이번달 지출(expense type)` - 고정저축 달성액
- 저축 달성액 = 고정저축 달성액 + max(추가 절약분, 0)

저축 목표 입력창 안내 멘트:
> "고정 저축이 먼저 반영되고, 수입에서 지출을 뺀 추가 절약분도 합산돼요"

### 9. AI 분류 오류 디버깅
- `api/ai-input/route.ts`에 상세 에러 로깅 추가
- Vercel 함수 로그에서 직접 오류 확인
- 원인 후보: API 키 미적용 / Gemini 응답 파싱 실패 / Rate limit

## 계정/로그인 로직 개선 (B안)

### 현황 문제
- 같은 이메일의 이메일/비밀번호 계정 ↔ 구글 계정이 자동 머지됨
- 보안 취약점 가능성 (Supabase Auth 설정 문제)

### 변경 방향

**자동 머지 비활성화:**
- Supabase 대시보드: Authentication → Settings → 이메일 자동 링크 옵션 확인 및 비활성화
- 충돌 시 "이미 이 이메일로 가입된 계정이 있어요. 해당 방법으로 로그인 후 연결해주세요" 안내

**명시적 연결만 허용:**
- 설정 탭에 "Google 계정 연결하기" 버튼 추가 → `supabase.auth.linkIdentity({ provider: 'google' })`
- 게스트 유저 → 구글 연결 → 데이터 유지 (같은 user_id)

**충돌 에러 처리:**
- `AuthError: User already registered` → 커스텀 안내 메시지
- `AuthError: Identity is already linked` → "이미 연결된 계정이에요" 메시지

### 온보딩 수정
- 이메일 회원가입 유저 + 게스트 유저 모두 온보딩 표시
- 현재 `guide_completed` 필드 기준으로 체크하는데, 신규 가입 시 항상 `false`로 초기화 확인

### 게스트 설정 화면
- 설정 탭에 게스트 전용 섹션 추가:
  - "📱 Google로 계속하기" 버튼 (계정 연동 = 데이터 유지)
  - 안내: "지금 연동하면 데이터가 유지돼요"
